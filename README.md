# CS-320
Software test Automation QA  
How can I ensure that my code, program, or software is functional and secure?  
I can ensure that my software is functional and secure by utilizing testing and defensive programming. Writing unit tests early, test early, test often, helps to verify that each method behaves as intended throughout the coding process. I use boundary testing, input validation, and exception handling to protect the system from invalid data and unexpected states. Security comes from minimizing assumptions, validating all external inputs, and designing code so that failures are predictable. Automated tests act as a safety net that catches regressions, while clear, modular code makes it easier to identify and fix potential vulnerabilities.
